## 野球部運用システム導入戦略提案書

### 1. 提案概要
- **目的**：LINEグループ中心の運用を脱し、各種業務（事務局・会計・記録・企画）を一元管理できる新システムへ移行することで、情報鮮度と共有効率を高める。
- **提案者**：〇〇（1年目／担当役割を記載）
- **提案審議先**：次回総会

### 2. 背景と現状課題
- **チャット偏重の限界**
  - 月初に配布されるスケジュールが、勝敗や天候による変更をリアルタイムに反映できない。
  - LINEは時系列表示のみで、2週間後の予定が最新投稿として流れ、誤解や見落としが発生する。
- **出欠管理の手作業負担**
  - ノートのコメント欄で出欠を収集し、事務局が手動集計。抜け漏れ確認やリマインドに時間がかかる。
- **会計処理の属人化**
  - 立替レシートをノート共有→会計担当が手入力で帳簿作成。履歴追跡が困難で、精算遅延や確認作業が増大。
- **資料・記録の散逸**
  - スコア表や資料、写真がLINEのノート・アルバムに散在し、年度を跨ぐとアクセスしにくい。
  - 毎年グループが新設されるため、過去データが埋もれやすい。

### 3. 新システムの概要
- **スタッツ・試合記録管理**
  - 対戦カード、スコア、個人成績を蓄積。過去シーズンの閲覧・検索が容易。
- **スケジュール管理**
  - カレンダー形式で予定を一元化。勝敗・天候による変更も即時反映。
  - 学年別・担当別のフィルター、通知設定に対応。
- **出欠・参加フロー**
  - イベントごとに出欠フォームを生成し、自動集計。未回答者リストや期限前リマインドを提供。
- **会計サポート**
  - 立替申請フォーム化、電子レシート添付。支払状況を一覧で管理し、月次レポートを作成。
- **資料・メディア共有**
  - 写真・資料をフォルダ単位で整理。年度を跨いでも検索可能。
- **アクセス権限管理**
  - 管理者、コーチ／OB、部員など役割に応じたロール設定。テストアカウントは既に稼働済み。

### 4. 期待効果
- **事務局の工数削減**
  - 出欠集計時間：現行1イベントあたり数十分 → 自動化でほぼゼロ。
  - スケジュール修正連絡の個別対応が不要となり、最新状態を常に共有可能。
- **会計処理の透明化**
  - 立替精算フローが統一され、履歴を即座に参照できる。帳簿作業時間を大幅縮減。
- **情報共有の正確性向上**
  - 最新の日程・資料にアクセスしやすく、連絡漏れや認識違いを減少。
- **データ資産化**
  - 成績・資料を継続蓄積し、次年度以降の分析・施策立案に活用可能。

### 5. 導入スケジュール案
1. **総会承認（○月）**  
   - デモ実施、質疑応答。
2. **パイロット運用（○月〜○月）**  
   - 事務局・会計チームで限定運用、課題・改善点を洗い出す。
3. **本格展開（○月）**  
   - 全員アカウント発行、初期トレーニング。一定期間はLINEと並行運用。
4. **定着化フェーズ（○月以降）**  
   - 月次レビューで改善を継続。翌年度の運用ルールを整備。

### 6. 体制・役割分担
- **システム責任者（提案者）**
  - 初期設定、権限管理、トレーニング実施。運用マニュアル整備、問い合わせの一次対応。
- **副管理者候補**
  - 自分の卒団・離任後の継続運用を見据え、学年を跨いで複数名を指名（例：事務局高学年担当1名、会計担当1名）。
  - 半年単位で役割交代を行い、ナレッジを共有する体制を構築。
- **事務局担当**
  - スケジュール登録・更新、出欠確認。
- **会計担当**
  - 立替精算承認、会計レポート管理。
- **記録担当**
  - スタッツ入力・更新、分析結果の共有。
- **企画担当**
  - 写真・動画・資料の整理とアップロード。

### 7. コスト・運用条件
- **システム利用料**
  - 現状はSupabase無料枠で運用（活動実績として定期ログインを継続）。
  - 利用拡大で有料化が必要な場合は月額○○円程度（試算）。予算計上は事前相談。
- **導入時の稼働**
  - 初期セットアップ：約○○時間（データ移行・権限設定・マニュアル作成）。
  - パイロット期間：週次レビュー30分程度。
  - トレーニング：全体説明会（60分）＋部門別ハンズオン。

### 8. リスクと対応策
- **ITリテラシー差への対応**
  - 画面キャプチャ付きマニュアル、短時間の動画デモを用意。問い合わせ窓口を明確化。
- **データ移行負荷**
  - 直近年度分から段階的に移行。過去データはPDF化・スクラップ保存も併用。
- **無料枠の活動要件**
  - 管理者・副管理者が月次でログイン確認。必要なら有料プランへの早期移行を検討。

### 9. 総会での承認事項
- 新システムパイロット導入の承認
- 副管理者（継続運用メンバー）選定の承認
- 各担当（事務局・会計・記録・企画）からテスト参加メンバーの指名
- 必要コスト（有料化を想定する場合）の事前承認または検討枠設定

### 10. 今後のアクション
- 副管理者候補の打診と合意形成
- 既存データ（スケジュール、出欠、会計、記録、写真）の移行準備
- 画面デモ資料・マニュアルの整備
- 総会向けプレゼン資料の作成およびリハーサル
